## Allowed

### Phase 5 — Operator intelligence (read-only)

- Conflict detection upgrades, derived risk scoring, rule-based approval summaries
- Agent detail drawer, evidence hash verify UI, overview nine-question completeness
- All Phase 5 work remains GET-only; inferred values must be labeled `INFERRED` (never `AUTHORITATIVE`)

### Phase 6 — Controlled operations completeness

- Enable `MISSION_CONTROL_PHASE6_COMMANDS=enabled` for loopback/staging when implementing
- Allowlisted `scripts/ados-tools/*` adapters for:
  - approved validator runs
  - approved integration request creation
  - bounded review pickup of already-approved work
- Extend `/operations` UI for these actions with consequence previews
- Negative tests proving unapproved/expired/consumed actions are denied

### Phase 7 — Alerting & external ops hooks

- Enable `MISSION_CONTROL_ALERTS=enabled` when implementing
- Local alert rules + opt-in webhook notifier with redacted payloads
- Alert history / mobile digest UI
- Grafana **external** provisioning docs only (no bundled Grafana server)

### Phase 8 — Platform hardening & performance

- SSE bounded-delta ADR (design first; implement only if non-fabricating)
- Snapshot performance work, fleet aggregation polish, threat-model refresh, e2e coverage

## Prohibited (all V3 phases)

- Cursor acquiring or inheriting ADOS PRIMARY lease through Mission Control
- Direct `state/**` writes from Next.js `app/` / `components/` / `lib/` (except existing approved SQLite cache + ados-bridge spawn)
- Direct `Invoke-CursorAgent` / `Launch-Cursor` from app surfaces
- Silent production dispatch enablement
- Cross-member fleet mutation, approve, or dispatch
- Embedding secrets/paths in Prometheus metrics or alert payloads
- Bundling Grafana/Prometheus servers into the Mission Control image
- Storing owner private keys in this repository or container image
- Cloud LLM exfiltration of control-plane secrets/paths (rule-based summaries only for P5-03 MVP)

## Execution protocol

1. Implement one backlog task (or tightly related cluster) at a time.
2. Keep phase flags fail-closed by default.
3. Land evidence under `docs/evidence/phaseN-*` when a phase epic completes.
4. Do not start Phase 6 staging enablement until Phase 5 exit gate criteria are met unless owner explicitly overrides.
